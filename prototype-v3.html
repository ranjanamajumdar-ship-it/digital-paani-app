Core UX pattern

  • One card. No lists, no branches. The system decides the next step.
  • Pictograms-first; text is bilingual support, not the primary signal.
  • Sticky 5-button footer is always there — operators never feel trapped.
  • Listen / auto-advance / bilingual are visible affordances, not hidden menus.
